Feature

The basis is find+grep.

Contents of all files below a specified directory are examined. 
gren pattern [dir]
OK.
% pwd
/Users/ongaeshi/Documents/gren # Presently FileTest of position the text which is written is searched
% gren FileTest 
lib/gren/findgrep.rb:34:        next unless FileTest.file?(fpath)
lib/gren/findgrep.rb:39:        unless FileTest.readable?(fpath)
lib/gren/findgrep.rb:44:        @result.size += FileTest.size(fpath)
lib/gren/findgrep.rb:53:        @result.search_size += FileTest.size(fpath)
lib/gren/findgrep.rb:95:      FileTest.directory?(fpath) &&

dir   : /Users/ongaeshi/Documents/gren (0.0sec)
files : 8 in 18 (7.9KB in 18.52KB)
match : 1 files, 5 hit

The entry which the user specifies in minimum

The tool being whether binary or the text automatic, it judges

  1. The specification like *.rb and */*.cpp is not necessary
  2. It includes to the retrieval object even with the special extension like README and README.ja
  3. You do not inspect the content of .png and .exe.

The exclusion directory is configurated with default

  • *~, .svn, .git and CVS etc exclude from the search object

Convenient search result

The directory (Path) which starts search, the time (Time) when it caught, the number (Files) of files, file size (Size) and the number of files and the number (Matches) etc of hits which match are displayed.

The compatibility of grep

Basic display format maintains the same shape, as grep can use in place of grep with such as grep-mode of emacs.

When description below is added to .emacs, you can use in place of grep-find.

Mac OS X 
(global-set-key "\C-x\C-g" 'grep-find) ; your favarite key !! 
(setq grep-find-command "gren ")
Windows 
(global-set-key "\C-x\C-g" 'grep-find) ; your favarite key !! 
(setq grep-find-command "gren.bat ")

Abundant option

It is possible to compact retrieval time, to squeeze the search object.
